# youtube-summary スキル設計

日付: 2026-07-08
ステータス: 承認済み(設計レビュー完了)

## 目的

YouTubeのURLを渡して要約を依頼すると、動画の文字起こし(字幕)を取得し、
聞き取りミス(誤認)を文脈から補正した上で、日本語の構造化要約をチャットに返すスキル。

## 決定事項(ユーザー承認済み)

| 項目 | 決定 |
|---|---|
| 字幕取得方法 | yt-dlp(未導入なら pip install してリトライ) |
| 出力形式 | チャットに要約のみ(ファイル保存なし) |
| 誤認補正 | 暗黙補正。確信が持てない重要語のみ「(原文: ○○)」注記 |
| 言語 | どの言語の動画でも日本語で要約 |
| 配置先 | skills-lab で開発 → 完成後 `~/.claude/skills/youtube-summary/` へコピー |
| アーキテクチャ | SKILL.md + 整形スクリプト(案B) |

## 構成

```
skills-lab/skills/youtube-summary/
  SKILL.md                      # フローと要約・補正の指示
  scripts/fetch_transcript.py   # 字幕取得・整形スクリプト(Python 3.12、標準ライブラリ + yt-dlp)
```

## コンポーネント

### scripts/fetch_transcript.py

決定論的な処理(取得・整形)をすべて担う。

- **入力**: 動画URL(必須)、`--out <dir>`(整形テキストの出力先、必須)、
  `--langs ja,en`(字幕言語の優先順、デフォルト `ja,en`)
- **URL対応形式**: `watch?v=` / `youtu.be/` / `shorts/` / `live/` / `embed/`。
  プレイリストURLは先頭(指定)の動画単体として扱う
- **字幕選択の優先順位**: 手動字幕(原語)→ 自動生成字幕(原語)→ 翻訳字幕
- **整形**: タイムスタンプ除去、自動字幕特有のロールアップ重複行の除去、プレーンテキスト化
- **出力**:
  - stdout にメタデータJSON: `title` / `channel` / `duration` / `language` /
    `subtitle_type`(manual|auto|translated)/ `transcript_path` / `char_count`
  - `--out` ディレクトリに整形済みテキストファイル
- **終了コード**: 0=成功、非0=失敗(stderr に原因: 字幕なし / yt-dlp未導入 /
  動画にアクセス不可 など、SKILL.md が分岐に使える形)

### SKILL.md

言語理解(補正・要約)を担う。フロー:

1. URLを受け取り、スクリプトをスクラッチパッド出力で実行
2. yt-dlp 未導入エラーなら `pip install yt-dlp` して1回だけリトライ
3. 整形テキストを読む。**8万文字超**なら offset/limit のチャンク読みで段階要約
   (チャンクごとに中間要約 → 最後に統合)
4. 誤認補正: 文脈から明らかな同音語・固有名詞の誤りは黙って直す。
   確信が持てない重要語のみ「(原文: ○○)」を付す
5. 日本語で要約を出力(テンプレは下記)。`subtitle_type` が auto なら冒頭に
   「自動生成字幕ベースのため固有名詞に誤りの可能性あり」と1行注記

要約テンプレ:

```markdown
## 要約: <動画タイトル>
(チャンネル名 / 動画の長さ / 字幕種別の注記)

**概要** 2-3文

**主要ポイント**
- 箇条書き(内容量に応じて3〜8点)

**結論・所感**
```

### description(トリガー)

日本語・英語のトリガー例を含める:
「このYouTube要約して」「この動画の内容教えて」+ YouTube URL、
"summarize this YouTube video"、URLのみ貼られて要約文脈がある場合。

## エラーハンドリング

- **字幕が一切ない**: その旨を報告して終了。音声ダウンロード→音声認識はスコープ外
  (将来拡張の余地としてここに明記)
- **非公開・年齢制限・地域制限**: yt-dlp のエラー内容を要約して報告
- **ネットワーク/yt-dlp 失敗**: リトライは1回まで。2回目の失敗で止めて報告(無限リトライ禁止)

## テスト計画(手動E2E)

1. 日本語動画(自動字幕)→ 日本語要約、auto注記あり
2. 英語動画 → 日本語要約
3. 字幕なし動画 → エラー報告(要約を捏造しない)

## スコープ外

- 音声認識による文字起こし(字幕なし動画への対応)
- 補正済み全文のファイル保存
- プレイリスト一括要約
